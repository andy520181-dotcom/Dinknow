# 快速创建数据库集合（解决活动发布失败）

## 问题原因
错误码 `-502005` 表示数据库集合不存在。从截图可以看到控制台显示"找不到可用的集合"，需要先创建集合。

## 立即执行（按顺序）

### 步骤 1：创建 `activities` 集合（活动信息）

1. 在 CloudBase 控制台 → **数据库** → **集合管理**
2. 点击搜索框右侧的 **"+"** 按钮
3. 输入集合名称：`activities`
4. 点击确认创建

**权限设置**（创建后立即设置）：
- 点击集合名称 `activities` → 右侧切换到 **"数据权限"** 标签
- **读取权限**：选择 `所有用户可读`
- **写入权限**：选择 `所有用户可写`（或使用表达式 `auth != null`）
- 点击保存

### 步骤 2：创建 `users` 集合（用户信息）

1. 再次点击 **"+"** 按钮
2. 输入集合名称：`users`
3. 点击确认创建

**权限设置**：
- 点击 `users` → **数据权限**
- **读取权限**：`所有用户可读`
- **写入权限**：`所有用户可写`（或 `auth != null`）
- 保存

### 步骤 3：创建 `registrations` 集合（报名记录）

1. 点击 **"+"** 按钮
2. 输入集合名称：`registrations`
3. 点击确认创建

**权限设置**：
- 点击 `registrations` → **数据权限**
- **读取权限**：`所有用户可读`
- **写入权限**：`所有用户可写`（或 `auth != null`）
- 保存

## 验证

创建完成后，在 **集合管理** 页面应该能看到三个集合：
- ✅ `activities`
- ✅ `users`
- ✅ `registrations`

## 重新测试

1. 重新编译小程序：`npm run build:mp-weixin`
2. 清除缓存后重新进入"发起活动"页
3. 填写表单并点击"发布活动"
4. 应该能成功发布（不再出现 `-502005` 错误）

## 注意事项

- **环境确认**：确保在正确的云开发环境中创建集合（免费开发环境 vs 生产环境）
- **权限设置**：如果使用"仅创建者可写"，创建新文档时会失败（因为创建时文档还不存在，无法匹配权限表达式）
- **首次使用**：建议先设置为"所有用户可写"，确保功能正常后再根据需要调整权限

## 如果仍有问题

1. 检查云函数 `createActivity` 是否已部署
2. 检查云开发环境 ID 是否与代码中 `src/constants.ts` 的 `CLOUD_ENV` 一致
3. 查看控制台日志中的 `[createActivity]` 日志，确认具体错误信息
