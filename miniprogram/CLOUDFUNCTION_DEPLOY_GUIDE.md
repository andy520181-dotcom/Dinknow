# 云函数部署操作指引

## 📦 部署 createActivity 云函数（添加球馆名称功能）

### 方法一：通过微信开发者工具部署（推荐）

#### 步骤 1：打开微信开发者工具
1. 打开 **微信开发者工具**
2. 导入项目：
   - 选择项目目录：`dist/build/mp-weixin`
   - 或直接打开已存在的项目

#### 步骤 2：定位云函数文件夹
1. 在项目文件树中找到 `cloudfunctions` 文件夹
2. 展开 `cloudfunctions` 文件夹
3. 找到 `createActivity` 文件夹

#### 步骤 3：上传并部署云函数
1. **右键点击** `createActivity` 文件夹
2. 选择菜单项：**"上传并部署：云端安装依赖"**
   - ⚠️ 注意：选择 **"云端安装依赖"** 而不是 "不上传 node_modules"
   - 这样会自动在云端安装 `package.json` 中的依赖（如 `wx-server-sdk`）

#### 步骤 4：等待部署完成
- 部署过程中，控制台会显示进度信息
- 看到 "上传成功" 或 "部署成功" 提示即完成

#### 步骤 5：验证部署状态
1. 点击工具栏的 **"云开发"** 按钮
2. 进入 **"云函数"** 页面
3. 在云函数列表中查看 `createActivity`
4. 确认状态显示为 **"已部署"** ✅

## 🔄 如果部署失败

### 常见问题 1：上传失败
**错误信息**：`上传失败` 或 `网络错误`

**解决方案**：
1. 检查网络连接
2. 确认微信开发者工具已登录
3. 重新右键上传

### 常见问题 2：依赖安装失败
**错误信息**：`npm install failed` 或 `依赖安装失败`

**解决方案**：
1. 确认选择了 **"云端安装依赖"**
2. 检查 `cloudfunctions/createActivity/package.json` 文件是否存在
3. 手动检查依赖配置：
   ```json
   {
     "name": "createActivity",
     "version": "1.0.0",
     "main": "index.js",
     "dependencies": {
       "wx-server-sdk": "~2.6.3"
     }
   }
   ```

### 常见问题 3：云函数调用失败
**错误信息**：`cloud.callFunction:fail` 或 `云函数不存在`

**解决方案**：
1. 确认云函数已部署（状态为"已部署"）
2. 确认云开发环境ID正确
3. 检查小程序代码中的云函数调用：
   ```typescript
   // 确认云函数名称正确
   wx.cloud.callFunction({
     name: 'createActivity',  // ✅ 确认名称正确
     data: { ... }
   })
   ```

## 💡 提示

- **部署时间**：通常 10-30 秒即可完成
- **无需重启**：部署后无需重启小程序，重新编译即可生效
- **版本管理**：每次部署会覆盖之前的版本，如需保留历史版本，可在云开发控制台查看版本历史
- **其他云函数**：如果其他云函数也需要更新，可以批量部署（依次右键上传）
