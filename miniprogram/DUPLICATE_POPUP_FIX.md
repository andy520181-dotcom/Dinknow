# 重复位置授权弹窗问题修复

## 🔍 问题分析

用户反馈：**出现两次位置授权弹窗**

### 可能的原因

1. **自定义隐私协议弹窗**（第一个）
   - 我们创建的自定义弹窗
   - 包含"用户协议"和"隐私保护"链接
   - 用户需要勾选并同意

2. **微信系统位置授权弹窗**（第二个）
   - 调用 `uni.getLocation()` 时自动弹出
   - 系统级别的授权弹窗
   - 必须通过此弹窗才能获取位置

---

## ✅ 已修复的问题

### 1. 防止重复调用位置API

添加了 `locationRequested` 标志，确保 `requestLocationAndLoad()` 只被调用一次：

```typescript
const locationRequested = ref(false) // 防止重复请求位置

async function requestLocationAndLoad() {
  // 防止重复调用
  if (locationRequested.value) {
    return
  }
  locationRequested.value = true
  // ... 请求位置逻辑
}
```

### 2. 优化初始化逻辑

- 已同意协议：延迟800ms后调用位置API
- 未同意协议：显示自定义弹窗，用户同意后延迟500ms再调用位置API
- 避免与页面加载冲突

### 3. 优化位置标签点击

- 如果已经请求过位置且有位置信息，不再重复请求
- 防止用户多次点击导致重复弹窗

## 📋 当前流程说明

### 用户首次打开小程序

1. **页面加载** → 检查是否已同意协议
2. **未同意** → 显示自定义隐私协议弹窗
3. **用户操作**：
   - 点击"用户协议"或"隐私保护指引"查看详情
   - 勾选"已阅读并接受《用户协议》和《隐私保护指引》"
   - 点击"允许"
4. **同意后** → 保存同意状态，调用位置API
5. **微信系统弹窗** → 弹出位置授权弹窗（用户授权位置）

### 用户再次打开小程序

- **已同意协议** → 直接调用位置API
- **不再显示自定义弹窗**
- **只显示微信系统位置授权弹窗**（如果之前拒绝了位置授权）

## ⚠️ 关于两个弹窗的说明

### 为什么会有两个弹窗？

这是**正常的设计流程**：

1. **第一个弹窗（自定义）**：
   - 目的：让用户阅读并同意"用户协议"和"隐私保护指引"
   - 包含可点击的协议链接
   - 符合隐私保护要求

2. **第二个弹窗（微信系统）**：
   - 目的：获取用户的位置授权
   - 系统级别的授权，必须通过此弹窗
   - 如果基础库 >= 3.4.2，此弹窗会自动附带隐私协议勾选框

### 如果只想显示一个弹窗

**方案1：移除自定义弹窗（不推荐）**
- 只使用微信系统的位置授权弹窗
- 但无法显示"用户协议"和"隐私保护"的详细链接
- 用户体验不够好

**方案2：保持当前设计（推荐）**
- 自定义弹窗：展示协议内容，用户明确同意
- 系统弹窗：获取位置授权
- 两个弹窗各有用途，符合规范
