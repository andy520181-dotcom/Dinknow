# 基础库下载失败问题 - 解决方案

## 问题描述

微信开发者工具提示：**下载基础库版本 3.14.2 失败：aborted**

## 原因分析

1. **基础库版本 3.14.2 是灰度版本**，可能不稳定或下载有问题
2. **miniprogramRoot 配置错误**：之前设置为 `dist/build/mp-weixin/`，应该是 `dist/`

## 已修复的内容

### 1. 修改基础库版本为稳定版本

**修改前：**
- `libVersion: "3.14.2"` （灰度版本）

**修改后：**
- `libVersion: "3.0.0"` （稳定版本）

### 2. 修正小程序根目录配置

**修改前：**
- `miniprogramRoot: "dist/build/mp-weixin/"`

**修改后：**
- `miniprogramRoot: "dist/"`

## 下一步操作

### 1. 重启微信开发者工具

完全关闭微信开发者工具，然后重新打开项目。

### 2. 重新编译

在终端运行：

```bash
cd /Users/andy/Desktop/Dinknow/miniprogram

# 清理旧的编译产物
rm -rf dist

# 重新编译
npm run dev:weapp
```

### 3. 在微信开发者工具中

1. 打开项目：`/Users/andy/Desktop/Dinknow/miniprogram`
2. 点击"编译"按钮
3. 如果还有问题，点击"清缓存" → "清除文件缓存"

### 4. 如果还是失败

可以在微信开发者工具中手动切换基础库版本：

1. 点击工具栏的"详情"
2. 找到"本地设置"
3. 在"调试基础库"中选择一个稳定版本（如 3.0.0 或 2.33.0）

## 其他可用的稳定基础库版本

如果 3.0.0 还有问题，可以尝试以下稳定版本：

- `2.33.0` - 较旧但非常稳定
- `2.34.0` - 较新的稳定版本
- `3.0.0` - 推荐的稳定版本

## 修改方式

如果需要手动修改，可以编辑以下文件：

**project.config.json**
```json
{
  "miniprogramRoot": "dist/",
  "libVersion": "3.0.0"
}
```

**project.private.config.json**
```json
{
  "libVersion": "3.0.0"
}
```

## 总结

主要问题是基础库版本选择了灰度版本 3.14.2，现在已经改为稳定版本 3.0.0。同时修正了小程序根目录配置。重启开发者工具并重新编译即可。
